# AIを活用しやすい機能まとめ

見積書などに記載されている「機能一覧」をベースとして、AI設計・実装のタスクとすべきものをまとめます。

## 見積書などの「機能一覧」から最初にAI向けタスクにするべき機能

| カテゴリ | 指示が簡単な理由 | 難易度
| :--- | :--- | :--- |
| **データ一覧・詳細表示** | 「入力パラメーター（検索条件）」と「出力プロパティ」の定義が明確なため。<br />TypeScriptの型定義やDTOを渡すだけでロジックが完結します。 | 低 |
| **データ登録・編集** | スキーマ（ZodやJSON Schema）を与えれば、UIからバックエンドの保存処理まで一貫して生成可能です。 | 低 |
| **帳票出力・エクスポート機能** | PDF出力、CSVエクスポート | 低 |
| **画面通知機能** | トースト表示、プッシュ通知のトリガー、確認ダイアログなど | 低 |
| **データ一括登録・更新** | CSVインポート処理、一覧画面からの直接編集機能など。<br />データ登録・更新のロジックを利用して実装可能です。<br />旧アプリなどが存在し、新アプリへのデータ置き換えなどがある場合はAI実装を検討に含めます。 | 中 |
| **ファイルアップロード/ダウンロード** | S3への登録、参照URL取得(開発環境と本番環境の分離が必要となる) | 中 |
| **認証・認可処理<br />ユーザー登録・管理** | 典型的な実装例は巷にあふれているため作ることは簡単だが、エンジニアによる情報管理が必要(開発環境と本番環境の分離が必要となる) | 中 |
| **メール通知処理** | 実装は容易だが、実装者による管理が必要(開発環境と本番環境の分離が必要となる) | 中 |


### 難易度・低の基準
AIで実装することを即決しても問題ないもの

- 要件一覧・機能一覧の段階で端的に機能を説明でき、明文化できるもの
- 開発環境と本番環境に動作の差異が **ないもの**

### 難易度・中の基準
AI実装をすべきもの。ただし開発環境や必要な情報が整った後にAIタスク化するもの

- 要件一覧・機能一覧の段階で端的に機能を説明でき、明文化できるもの
- 開発環境と本番環境で動作に差異が **あるもの**
- 「難易度・低」の機能を下敷きとして実装を進めるもの
- 外部のデータ(例：旧アプリのエクスポートファイル)などに対応する処理
