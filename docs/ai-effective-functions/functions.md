### 高効果

| 機能 | 指示難易度 | 備考 |
|-|-|-|
| 100件程度までの一覧表示 | 極めて簡単|  データ構造またはスキーマが決まっているだけで「〇〇の一覧画面を作れ」くらいの指示で実装可能 |
| 100件超の一覧表示 | 簡単 | API側にページング機能を付ける必要がある |
| 100件超の一覧表示<br />（項目によるソートあり） | やや複雑 | API側にソート条件の指定とソート後にページングする機能を付ける必要がある。APIが過剰実装にならないように注意する必要がある |
| 1件の生成・更新 | 極めて簡単| upsertでAPIを実装して、新規登録画面を2つ開いていても冪等性を維持可能。 |
| 一括追加・編集 | やや複雑 | 一覧機能と1件生成・更新機能を組み合わせる。一覧に対象が100件未満の規模であれば容易 |
| 一般的な画面レイアウト<br />(ヘッダー/サイドバー/コンテンツ) | 簡単 |  |
