# docs\problem-02 開発ガイド (フロントエンド連携 & API同期)

このディレクトリのプロジェクトは,`specification.*` の定義から AI が API クライアント、エラーハンドリング、バリデーションを自動生成し、フロントエンドとバックエンドの同期を維持するスタイルを採用しています。

## 開発・ビルドコマンド
- ビルド: `npm run build`
- テスト: `npm test`
- APIクライアント生成: `npm run generate:api`
- lint修正: `npm run lint:fix`

## 実装規約 (AIへの指示)
フロントエンドとバックエンドの整合性を保つため、以下のルールを遵守してください。

### 1. APIクライアントの自動生成
- **全件対応:** バックエンドが公開しているすべての API エンドポイント（REST/GraphQL/WebSocket）に対応する API クライアントを生成する。
- **同期:** `specification.*` に変更があった場合、AI は即座に API クライアント、DTO、および関連するエラーハンドリングコードを更新する。

### 2. アクセス経路とセキュリティ
- **外部公開:** ブラウザ側から呼び出す API に対してのみ、外部アクセス経路（Endpoint）を構築する。
- **サーバー間通信:** サーバー内部でのみ使用される API クライアントは、ブラウザから直接呼び出せないよう隔離する。
- **認証:** 通信時は `Cookie` を使用せず、`JWT Bearer` トークンによる認証に置き換える。

### 3. 共通機能の自動組み込み
- **エラーハンドリング:** 標準的なリトライ・フォールバック、およびロギング・モニタリング機能を API 通信に自動で組み込む。
- **バリデーション:** 出力（レスポンス）バリデーションを実装し、定義通りのデータがフロントエンドに届くことを保証する。

### 4. テスト方針
- API クライアントの生成と同時に、モックサーバーを利用した通信テストコードを必ず生成する。
